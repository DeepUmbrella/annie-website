# Annie 全站 UI 质感精修设计

## 目标

在当前整站前台统一基础上，再做一轮全站 UI 质感精修，重点解决标题系统偏小、文字与背景对比不足、卡片缺少风格和页面视觉气场不够的问题。此次工作不是重新做页面结构，而是在现有统一骨架之上，把整站整体往“更大气、更清晰、更有设计感”的方向拉一档。

## 范围

本次包含：
- `Home`
- `Features`
- `Docs`
- `Blog`
- `Contact`
- `Login`
- `Register`
- 这些页面所依赖的公共标题、卡片、文本层级与表单视觉规则

本次不包含：
- Header / Footer 结构重做
- 页面信息架构重做
- 新功能开发
- 后台页面改版

## 用户反馈归纳

当前用户反馈集中在以下几点：
- 不只是 `Features`，其他页面也有问题
- 各种 `title` 都可以设计得更大一些
- 某些卡片里文字与背景颜色接近，不容易看清
- 页面虽然统一了，但设计感和风格感还不够强
- `Home` 页面也存在同类问题，不是只有内页

## 本轮结论

因此本轮不是某一页专项改版，而是：
**全站围绕标题系统、可读性、卡片风格和页面气场做一次横向精修。**

## 核心设计原则

### 1. 不重做结构，只提升质感

这轮不推翻现有页面结构。Hero、section、卡片、表单等布局骨架继续保留，重点放在：
- 标题大小
- 文本层级
- 对比度
- 卡片视觉个性
- 页面气场

### 2. 标题系统整体上提

整站当前最大的共性问题之一，是标题偏保守，导致页面不够大气。此次要统一提升：
- 页面主标题
- section 标题
- 卡片标题
- 表单页标题

要求不是简单“都放大”，而是让不同层级之间拉开明显差距。

### 3. 可读性优先于炫光

如果文字不清楚，再好的 glow 和玻璃感也没有意义。本轮必须优先修正：
- 标题与背景的对比
- 正文与次级文案的层级
- 卡片内部信息的识别效率
- 深色背景上的内容可读性

### 4. 卡片要从“统一”走向“有风格”

当前有些卡片已经统一，但还停留在“玻璃卡片模板”的阶段。下一步要让卡片具备：
- 更明确的信息层级
- 更强的标题锚点
- 更清楚的装饰或视觉识别
- 页面类型之间的差异化气质

## 需要精修的共性问题

## 1. 标题系统

### 页面主标题
要求：
- 更大
- 更有压迫感
- 更像产品官网标题
- 与副标题形成明显层级落差

适用页面：
- Home
- Features
- Docs
- Blog
- Contact
- Login
- Register

### Section 标题
要求：
- 明显撑起区块
- 不能过于保守
- 与 eyebrow 和描述之间有清晰节奏

### 卡片标题
要求：
- 不能和正文一样轻
- 应成为卡片内部第一识别点
- 标题字号、字重、颜色都应更稳

### 表单页标题
要求：
- Login / Register 的标题更像产品入口页，而不是普通表单页文案
- 在视觉上形成清晰中心

## 2. 文字层级与对比度

### 标题文字
- 主标题需保持更高亮度和更强权重
- 深色背景上的主标题不能发灰

### 正文与辅助文案
- 次级文案可以柔和，但不能弱到读不出来
- 辅助说明与正文之间要有明确区分

### 卡片内部文字
- 标题与描述必须拉开
- 某些页面卡片中的文字目前与背景太接近，需要提高对比度

## 3. 卡片风格精修

### 共性要求
所有卡片不再只是统一外壳，而要有更强的“内容组织感”：
- 标题更明确
- 描述更干净
- 视觉锚点更明显
- hover 与状态更自然

### 页面差异化要求
- `Home` / `Features`：卡片更具展示感与产品感
- `Docs` / `Blog`：卡片更规整、信息导向更强
- `Contact`：卡片更偏信任感和清晰入口
- `Login` / `Register`：表单卡更聚焦、更稳

## 页面级精修方向

## Home

问题：
- 标题和 section 仍可更大气
- 某些卡片信息层级还可以更明显
- 下半部分区块需要进一步拉开 section 标题和正文层级

方向：
- 强化主标题与 section 标题
- 强化能力区/场景区卡片标题与正文的层次
- 保持首页是整站最强视觉页面

## Features

问题：
- 卡片视觉存在同质化
- 标题与背景/卡片层次不够
- 页面仍像功能列表，而不是产品能力展示页

方向：
- 强化页面主标题与功能模块标题
- 让卡片更具展示感
- 提高文字对比与识别效率

## Docs

问题：
- 有可能出现风格统一了但可读性下降
- 搜索区、分类区标题层级可能还不够强

方向：
- 提升标题和搜索区权重
- 增强分类卡标题与说明区分
- 保证开发者文档入口清晰易读

## Blog

问题：
- 博客卡片可能统一了样式，但还不够像产品内容卡
- 标题与摘要层次可能仍不够拉开

方向：
- 强化文章标题锚点
- 提升摘要可读性
- 让博客页更像产品更新页，而不是普通文章列表

## Contact

问题：
- 联系方式卡片可能统一了，但还不够有设计感
- 信息入口感需要更明确

方向：
- 提升标题与联系入口清晰度
- 让卡片更像可信赖的产品联系入口

## Login / Register

问题：
- 标题和表单卡虽然统一了，但还可以更有“转化入口”气场
- 输入区与辅助说明的层级仍可优化

方向：
- 放大标题与品牌区
- 增强表单区聚焦感
- 让按钮与辅助文案更有产品感

## 验收标准

完成后应达到：
- 全站各类标题明显更大气
- 文字与背景对比更稳，阅读更轻松
- 卡片不再只是统一样式，而更有设计感与页面个性
- Home、Features、Docs、Blog、Contact、Login、Register 都同步提升
- 页面结构保持稳定，但视觉气场整体提升一档
