# Annie 首页与公共布局视觉改版设计

## 目标

将 Annie 前台站点的首页与公共头部/底部改造成更接近 openclaw.ai 的产品官网气质，重点借鉴其整体视觉语言与页面节奏，而不是做 1:1 复刻。

本次改版目标：
- 提升首页的产品感、现代感与技术感
- 统一 Header / Footer / Home 的视觉风格
- 引入 Tailwind CSS 作为主要布局与样式表达方式
- 保留部分 Ant Design 组件能力，但尽量降低 Ant Design 默认视觉痕迹

## 范围

本次仅包含以下内容：
- `frontend/src/components/common/Header.tsx`
- `frontend/src/components/common/Footer.tsx`
- `frontend/src/pages/Home.tsx`
- 前端 Tailwind 基础样式接入与最小设计 token

本次不包含：
- `Features`
- `Docs`
- `Blog`
- `Contact`
- `Login`
- `Register`
- 移动端复杂导航交互（例如抽屉菜单）
- 全站完整 design system

## 设计原则

### 1. 借鉴“气质”和“结构”，不做像素级复刻

目标是吸收 openclaw.ai 的这些特征：
- 深色背景上的高对比排版
- 克制的渐变与高光
- 清晰的 section 节奏
- 技术产品官网的冷静感与未来感
- 简洁但有层次的卡片与 CTA

不做以下事情：
- 不复刻 openclaw.ai 文案
- 不复制其插画、图形或品牌元素
- 不为了“像”而牺牲 Annie 自己的内容表达

### 2. Tailwind 为主，Ant Design 混用

样式实现策略：
- 布局、颜色、间距、圆角、边框、响应式主要由 Tailwind 完成
- Ant Design 仅在按钮等局部交互控件中保留，且需要通过 className 或外层容器弱化默认样式痕迹
- Home / Header / Footer 不应继续以大量 inline style 作为主要样式手段

### 3. 先做最小视觉 token，不做完整设计系统

这次不单独搭建大型 design system，但需要先确定一套最小可复用 token：
- 背景色
- 文本色
- 强调色
- section 容器宽度与间距
- 卡片样式
- 按钮样式

## 视觉方向

### 色板

建议使用以下主色：
- 背景主色：`#0B1020`
- 背景次色：`#11162A`
- 品牌紫：`#7C3AED`
- 品牌粉紫高光：`#A78BFA`
- 品牌青蓝点缀：`#22D3EE`
- 正文主色：`#F8FAFC`
- 正文次色：`#94A3B8`
- 边框色：`rgba(255,255,255,0.10)`
- 卡片底色：`rgba(255,255,255,0.04)`

使用规则：
- 页面以深色背景为主
- 高亮色只用于按钮、badge、局部 glow、图标点缀
- 不大面积使用高饱和渐变铺满整个页面

### 排版

字体使用：
- `Inter`
- `ui-sans-serif`
- `system-ui`

层级规则：
- Hero 标题：超大号、粗体、紧凑行高
- Section 标题：明显大于正文，但较 Hero 更克制
- 正文：中等字号、偏长行高、灰白色
- Eyebrow / Badge：更小字号，强调轻量标签感

### 卡片与按钮

卡片：
- 深色半透明背景
- 细边框
- 中到大圆角
- hover 时轻微上浮或边框提亮
- 标题用亮色，描述用次级文本色

按钮：
- Primary：紫色或紫蓝渐变底，白字，轻微发光感
- Secondary：透明底 + 边框，hover 时边框/背景略增强

## 页面结构设计

## Header

### 信息结构
- 左侧：Annie 品牌标识 / 名称
- 中间：`功能`、`文档`、`博客`、`联系`
- 右侧：`登录`、`注册` 或已登录状态

### 样式
- 顶部悬浮
- 内容居中，最大宽度限制
- 半透明深色背景
- 启用 `backdrop-blur`
- 使用较大圆角，形成产品官网常见的 floating nav 视觉

### 交互
- hover 时文字提亮
- 当前路由有轻量高亮态
- 本次不处理复杂移动端导航，仅保证桌面端主视觉成立

## Home

### Section 1: Hero

#### 左侧内容
- 小型 badge，例如 `AI Assistant Platform`
- 2 至 3 行主标题
- 一小段副标题，解释 Annie 的定位与价值
- 两个 CTA：
  - 主按钮：立即体验
  - 次按钮：查看文档

#### 右侧内容
- 一个主视觉容器
- 容器中放置 2 到 3 个浮层能力卡片，例如：
  - 对话能力
  - 知识支持
  - 自动化能力

#### 目标
Hero 的作用是建立第一印象，不展示后台截图，不堆砌文案，而是快速传达“Annie 是一个现代 AI 产品”。

### Section 2: 核心能力

建议使用 6 张卡片，分别对应：
- 智能对话
- 知识检索
- 自动化工作流
- 开发者接入
- 多端协作
- 可控与安全

每张卡片包含：
- 小图标或编号
- 标题
- 一句说明

布局建议：
- 桌面端 3 列
- 中等屏幕 2 列
- 小屏单列

### Section 3: 使用场景

使用 3 张更大的场景卡：
- 个人效率助手
- 团队协作中枢
- 开发者 AI 接入层

目标：
- 从“功能说明”转为“结果导向”表达
- 让页面更像成熟产品官网，而不是功能列表页

### Section 4: 底部 CTA

使用一个完整的收尾区块：
- 标题
- 一句辅助说明
- 两个 CTA

建议文案方向：
- 开始使用 Annie
- 查看开发文档

目标：
- 在页面结尾形成明确转化出口
- 与 Hero 首尾呼应

## Footer

### 信息结构
- 左侧：Annie 品牌说明
- 右侧：少量链接分组，例如：
  - 产品
  - 资源
  - 联系

### 样式
- 延续深色背景
- 文字层级更弱
- 边框更淡
- 留白更充足
- 不做复杂企业站式的大而全 footer

## 技术设计

### Tailwind 接入方式

需要完成以下基础接入：
- 安装 Tailwind CSS
- 补齐 Tailwind 所需配置文件
- 在全局样式入口挂载 Tailwind
- 定义最小可复用的主题扩展（颜色、阴影、圆角、容器）

### 样式组织原则

优先顺序：
1. Tailwind utility class
2. 少量可复用 class 抽取
3. 必要时保留极少量局部 CSS

避免：
- 大量 inline style
- 单页写死太多重复颜色值
- 同一视觉模式在多个组件中重复硬编码

## 可接受的实现结果

完成后应达到：
- 首页第一屏明显比当前版本更有产品官网感
- Header / Footer / Home 在视觉上属于同一套语言
- Ant Design 默认风格不再主导页面观感
- 后续扩展到 Features / Docs / Blog 时可以沿用同一套视觉 token

## 非目标

本次不要求：
- 完成所有页面统一改版
- 做完整响应式细节打磨到所有断点
- 建立完整组件库
- 复刻 openclaw.ai 的所有动画、图形或信息架构

## 验收标准

设计验收以以下标准判断：
- 首页视觉方向明显接近 openclaw.ai 的“产品官网气质”
- 页面结构包含 Hero、核心能力、使用场景、底部 CTA
- Header 与 Footer 风格统一
- Tailwind 已成为主要样式工具
- 代码中不再依赖大面积 inline style 来完成新首页视觉
