# Annie 全站前台统一设计

## 目标

将 Annie 当前前台页面统一到同一套 AI 产品官网视觉语言下，使首页、功能页、文档页、博客页、联系页以及登录注册页在气质、布局和组件风格上保持一致。目标不是让所有页面长得一样，而是让用户在全站切换时明显感受到这是一套统一的产品体验。

## 范围

本次包含：
- `Home`
- `Features`
- `Docs`
- `Blog`
- `Contact`
- `Login`
- `Register`
- 公共 `Header`
- 公共 `Footer`
- 支撑这些页面的公共布局与视觉原语

本次不包含：
- 后台页面
- 复杂动画系统
- 新插画资产或重型视觉资源
- 完整 design system 平台化建设

## 用户选择结论

用户已明确：
- 下一步希望优化其他页面
- 范围选择为“全站前台页面一次性统一”
- 实现方式选择为：先定义整站页面骨架，再将所有前台页面纳入同一风格
- 风格上选择：全部页面都按首页这套强视觉路线统一，包括登录注册页

因此本次方向为：
**整站统一强视觉，但不同页面按内容类型控制强度。**

## 设计原则

### 1. 统一语言，不强行统一页面长相

全站需要统一：
- 颜色体系
- 页面容器
- 标题层级
- 卡片风格
- 表单风格
- CTA 风格
- 背景与 glow 语言

但不同页面仍需保留不同信息密度与视觉强度。

### 2. 首页最强，但不能形成断层

首页依然是整站视觉最强的页面。

但 Features、Docs、Blog、Contact、Login、Register 不能像两个完全不同的网站。整站应呈现“同一家产品”的连续体验。

### 3. 先统一骨架，再统一页面

优先建立：
- 页面 Hero 骨架
- Section 标题模式
- 内容卡模式
- 表单卡模式
- 空状态/列表样式

然后将这些模式应用到所有页面，而不是每页随意发挥。

## 页面分型

## A. 营销展示页
包含：
- `Home`
- `Features`
- `Contact`

特点：
- 视觉强度最高
- Hero 更明显
- CTA 更突出
- 背景 glow 更完整
- 卡片更具产品展示感

## B. 内容信息页
包含：
- `Docs`
- `Blog`

特点：
- 仍保留产品风格
- 但优先保证信息可读性
- 列表、卡片、搜索和分类要更规整
- 不应为了炫而牺牲使用效率

## C. 表单转化页
包含：
- `Login`
- `Register`

特点：
- 依然保留强视觉路线
- 但信息结构更聚焦
- 用中心卡片承载主要操作
- 更像高级产品的转化入口页

## 整站统一骨架

### Header / Footer
继续沿用当前已建立方向：
- Header：悬浮式、深色、科技产品官网气质
- Footer：深色、克制、结构清晰

这两者作为全站壳层保持统一，不再每页单独变化。

### 页面容器
所有页面统一：
- 最大宽度
- 横向 padding
- section 纵向间距
- 进入页面后的视觉节奏

### Section 标题模式
统一使用：
- eyebrow（小标签）
- 主标题
- 辅助说明

即使不是所有页面都需要完整三层，也应尽量遵循这个模式，保持页面语言统一。

### 卡片模式
统一的内容卡应具备：
- 深色半透明底
- 细边框
- 中到大圆角
- hover 时轻微抬起或提亮
- 标题白色，辅助描述灰白色

### CTA 模式
全站 CTA 保持：
- Primary：高亮渐变主按钮
- Secondary：玻璃/描边按钮

不同页面允许强弱不同，但不允许出现新的风格分叉。

## 页面级设计

## 1. Features

### 目标
从普通功能页升级为产品能力展示页。

### 页面结构
- Hero：功能概览与产品定位
- 功能模块：用统一卡片展示核心能力
- 优势区：突出 Annie 的差异点
- CTA 收尾

### 风格要求
- 与 Home 保持高一致性
- 比 Home 略克制，但仍明显有产品展示感

## 2. Docs

### 目标
做成开发者文档入口页，而不是普通内容页。

### 页面结构
- Hero：开发者文档定位说明
- 搜索区：搜索入口清晰突出
- 文档分类区：卡片化分类导航
- 热门内容或推荐入口

### 风格要求
- 保留深色和科技感
- 但优先保证检索、分类、可读性
- 不应让视觉效果干扰文档导航效率

## 3. Blog

### 目标
做成更像产品博客/产品更新页。

### 页面结构
- Hero：博客简介
- 文章列表卡片
- 标签/分类或精选内容
- 空状态样式优化

### 风格要求
- 卡片统一
- 列表更规整
- 与首页保持气质一致，但弱化首页级强视觉冲击

## 4. Contact

### 目标
做成现代产品联系页，而不是传统企业站联系页。

### 页面结构
- Hero：联系引导
- 联系方式卡片
- 联系/咨询入口
- 表单或渠道说明

### 风格要求
- 保留产品感与信任感
- 避免传统“联系我们”页面的老式视觉

## 5. Login / Register

### 目标
做成整站统一的深色高质感转化页。

### 页面结构
- 页面中心为主表单卡
- 上方保留品牌与清晰标题
- 下方为切换链接和辅助说明

### 风格要求
- 背景延续整站 glow 语言
- 表单卡更精致、更聚焦
- 输入框、按钮、错误提示风格统一
- 仍然强视觉，但不应花过首页 Hero

## 技术实现约束

- 继续以 Tailwind 为主要样式表达方式
- 尽量减少 inline style
- 继续复用和扩展现有公共组件
- 在必要时新增最小公共页面原语，但避免无关抽象
- 不为每个页面单独发明一套样式模式

## 验收标准

完成后应达到：
- 全站前台页面视觉上明显属于同一套产品语言
- 首页最强，但其他页面不会显得像不同网站
- Features / Contact 更有展示感
- Docs / Blog 更规整、更可读
- Login / Register 更像高级产品转化页
- Header / Footer 作为全站壳层持续稳定工作
- 构建通过，页面改动范围保持在前台页面与公共布局层
