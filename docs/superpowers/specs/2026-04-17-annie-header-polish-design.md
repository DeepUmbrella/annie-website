# Annie Header 精修设计

## 目标

只针对 Annie 首页与站点公共 Header 做一轮小范围精修，解决当前 Header 在品牌识别、导航气场与右侧操作区视觉权重上的不足。

## 范围

仅包含：
- `frontend/src/components/common/Header.tsx`
- 若确有必要，可微调 `frontend/src/components/common/ButtonLink.tsx` 中 Header 复用到的按钮视觉，但不扩大到首页其他区块

本次不包含：
- Home 其他 section 改动
- Footer 改动
- 页面结构重做
- 移动端导航交互扩展

## 用户反馈要点

用户明确提出：
- 左上角 `Annie AI` 太小，不够明显
- 中间功能区选项按钮太小，不够大气
- 登录区应更明确地位于右侧，并具备更完整的视觉存在感

## 设计结论

### 1. 左侧品牌放大并增强锚点感

目标：
- 让品牌成为 Header 中最稳定的第一视觉锚点
- 提升“Annie AI”被一眼记住的概率

调整方向：
- 字号更大
- 字重更强
- 字距略微收紧或轻微优化
- 可以加入轻量高亮，但不做夸张渐变 logo

### 2. 中间导航更大气

目标：
- 不再只是普通文本链接
- 更像产品官网导航，而不是页面内普通菜单

调整方向：
- 导航字号适度增大
- 链接之间间距拉开
- 默认态更克制，但 hover/active 更明确
- 整体视觉应显得更“稳”和“开阔”

### 3. 右侧登录区更像独立操作区

目标：
- 明确 Header 的右侧是操作区，而不仅是几个附带按钮
- 提升右侧区块的完整性和平衡感

调整方向：
- 登录和注册按钮作为一个独立区块存在
- 登录按钮保持轻量，但不应弱到几乎没有存在感
- 注册按钮保持主按钮地位
- 已登录状态下，用户名/退出也要保留同样的右侧操作区结构感

## Header 整体平衡要求

精修后应满足：
- 左侧品牌、中间导航、右侧操作区三者权重更平衡
- Header 更像完整的产品官网顶部栏
- 视觉上更大气，但不变厚重
- 仍保留当前已经建立的未来感与悬浮感

## 验收标准

- `Annie AI` 一眼更明显
- 中间导航明显更大气
- 右侧登录区更完整、更靠右、更像操作区
- Header 不因放大元素而变得拥挤或失衡
