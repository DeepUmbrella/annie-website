# Annie 首页视觉精修设计

## 目标

在第一轮首页改版基础上，继续将 Annie 首页、Header 和 Footer 的视觉表现往更强未来感、更高冲击力的方向推进。此次工作不再调整页面基本结构，重点提升气质、光效、排版张力与科技产品氛围，使整体更接近 openclaw.ai 所体现的视觉能量。

## 范围

本次仅包含：
- `frontend/src/components/common/Header.tsx`
- `frontend/src/components/common/Footer.tsx`
- `frontend/src/pages/Home.tsx`
- 支撑这些视觉精修所需的最小样式 token 调整

本次不包含：
- 页面信息架构重做
- Features / Docs / Blog / Contact / Login / Register 改版
- 大规模动画系统
- 新增复杂 3D 图形或插画资产

## 方向结论

用户希望这次精修整体偏向：
- 更炫一点
- 更未来感
- 更有视觉冲击力

但边界明确：
- 不做赛博朋克海报风
- 不堆叠过量渐变
- 不牺牲可读性与产品官网感

所以本次方向定义为：
**高科技感产品官网，而非夸张视觉实验页。**

## 核心设计原则

### 1. 结构不变，视觉提档

Hero、能力区、场景区、CTA 区的结构保持不变，避免再次扩大实现范围。所有精修围绕这些现有区块展开。

### 2. Hero 优先，整页同步提档

虽然整页都会调整，但重点放在 Hero：
- 标题张力
- 按钮主次关系
- 右侧视觉面板
- 背景光效

整页其余 section 同步提档，确保不是“只有第一屏好看”。

### 3. 发光感来自层次，不来自滥用颜色

本次未来感主要通过以下手段获得：
- 深色背景上的局部高亮
- 柔和的紫蓝 glow
- 半透明面板与描边
- 模糊、分层、浮动关系
- 更清晰的排版层次

而不是通过：
- 整页高饱和霓虹色
- 复杂杂乱的背景
- 多种强调色同时抢视觉中心

## 视觉精修方向

## Header

### 目标
让 Header 更像悬浮在页面顶部的“控制条”，而不是普通导航栏。

### 调整内容
- 缩薄整体高度，减少厚重感
- 增强悬浮感和漂浮感
- 背景更通透，弱化实体盒子感
- 边框更细、更淡
- hover 状态提亮更明显
- 登录/注册按钮与 Header 主体形成更清晰主次

### 结果要求
用户第一眼看 Header，会觉得它属于 AI 产品官网，而不是通用后台导航。

## Hero

### 目标
将 Hero 打造成整页最强视觉焦点。

### 标题
要求：
- 更大
- 更紧凑
- 更少废话
- 某些关键词可做渐变高亮

标题不追求很长，而追求有力度、有压迫感。

### 副标题
要求：
- 比当前更短
- 更像产品定义句
- 保持高可读性

### CTA
要求：
- 主按钮更亮、更有能量感
- 次按钮更像玻璃质感按钮
- 主次关系更清晰
- hover 态更明显

### 右侧视觉面板
目标不是展示真实后台截图，而是构建 AI 平台感：
- 一个主面板
- 2 到 3 个浮层卡片
- 面板内部信息块更像“能力模块”或“智能面板”
- 用 glow、模糊、描边、分层营造未来感

这一区域的作用是让用户在不看文案时，也能感知“这是一个 AI 产品”。

## 页面背景与光效

### 目标
增强页面的空间感和品牌氛围。

### 调整内容
- Hero 顶部加入更大范围的柔和紫蓝 glow
- 页面中引入局部光斑、模糊背景或弱网格感
- section 之间通过明暗和透明层次形成更自然过渡

### 边界
所有背景光效必须是“柔和支撑”，不能抢正文内容的注意力。

## 核心能力卡片

### 目标
从“功能说明卡”升级为“科技面板卡”。

### 调整内容
- 更轻的卡体
- 更细的描边
- 更明显的 hover 提升
- 加入小型发光点、图标容器或编号装饰
- 标题和描述的层级更明确

### 结果要求
卡片看起来像产品能力模块，而不是普通营销网站的说明块。

## 使用场景区

### 目标
让场景区更像解决方案展示，而不是普通三栏列表。

### 调整内容
- 每张卡更大、更松弛
- 留白更多
- 文案更克制
- 卡片气质更高级
- 必要时加入轻量高光或面板层叠

## CTA 收尾区

### 目标
让页面结尾形成明确聚焦与能量收束。

### 调整内容
- CTA 区更像一个聚焦能量块
- 背后可有柔和 glow
- 标题更有号召力
- 按钮视觉更强
- 与 Hero 在气质上首尾呼应

## 颜色与样式微调

### 颜色策略
保留第一轮已建立的深色 + 紫蓝系方向，但进一步加强：
- 紫色与青蓝的局部互动
- 按钮高光更明确
- 边框对比略增强
- 文字层级更清楚

### 圆角与阴影
- Header 与按钮继续保持较大圆角
- 卡片圆角统一但不要过分软萌
- 阴影从“厚重投影”转向“柔和发光 + 层次分离”

## 技术实现约束

- 继续以 Tailwind utility class 为主
- 不重新引入大量 inline style
- 不增加复杂动画库
- 不引入额外视觉资源依赖，优先用 CSS 和现有结构完成未来感增强
- 改动尽量集中在 Home / Header / Footer 与必要 token 扩展

## 验收标准

完成后应达到：
- 整页比第一轮版本更有未来感和视觉冲击力
- Hero 明显成为主视觉中心
- Header 更轻、更浮、更高级
- 卡片与 CTA 更像科技产品官网组件
- 页面整体更接近 openclaw.ai 的视觉能量，而不是只停留在深色换肤层面
- 仍保持可读性、结构清晰度和部署可用性
